How it works

  • Pick a model tab and fill in the sections that apply — optional ones are labeled, so a quick prompt can skip them.
  • Each builder follows that model's best practices, so the same idea gets shaped differently for Claude, Gemini, and ChatGPT.
  • Attach reference files by uploading them. Browsers only expose the file name, so the generated prompt asks the model to locate each file by searching for it by name.
  • Copy grabs the assembled prompt; Reset clears the form.
  • Runs entirely in your browser. No API calls, no accounts, nothing sent anywhere — it just assembles text. Free to use and share.

📎 A note about uploaded files

This tool only reads the names of files you attach — never their contents or their location on your computer (browsers don't allow that). What you actually get is a prompt that references those files by name. What happens next depends on where you paste it:

  • In a regular chat (Claude.ai, Gemini, ChatGPT, etc.): the assistant can't reach into your computer, so when you send the prompt it will likely ask you to attach the files again. Just upload them from your machine at that point.
  • In an agent system connected to your machine (Claude Code, Codex, and similar): it can act on the instruction and go search your machine for each file by name and read it for you — no re-uploading needed.

Claude Prompt Generator

The anatomy of a Claude prompt — fill in each section, then copy the assembled brief.

Task

What you're working on, who it's for, and the specific ask.

Context Files

Files to read completely before responding. Upload to grab names, then edit the path and say what each contains.

Reference

A reference for what you want to achieve.

Effort

Scope it like it's at the top of your range — don't undersell.

Act

When you have enough info to act, act. Don't re-litigate decisions or survey options you won't pursue.

Scope

Do the simplest thing that works well. No extra features, refactors, or abstractions.

Delegate

Split independent subtasks across subagents. Verify with a fresh-context subagent against the spec.

Evidence

Before reporting progress, audit every claim against a tool result. Unverified? Say so. (Always included.)

Memory

Record lessons — one per file, one-line summary. Update, don't duplicate.

Checkpoint

Pause only for: destructive actions, real scope changes, or input only you can provide. (Always included.)

Report

Open with the outcome — the TLDR. Complete sentences. Clear beats short. (Always included.)
